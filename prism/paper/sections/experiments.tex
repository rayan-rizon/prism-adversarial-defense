\section{Experiments}
\label{sec:experiments}

We evaluate \prism on CIFAR-10 and CIFAR-100 against white-box, black-box,
optimization-based, and adaptive attacks, followed by ablation, campaign,
recovery, and baseline comparisons.

% ── Setup ──────────────────────────────────────────────────────────────────
\subsection{Experimental Setup}
\label{sec:setup}

\paragraph{Base classifier.}
We use a CIFAR-native ResNet-18 \cite{he2016deep} trained from scratch on
the active dataset.
\prism monitors activations at $L=3$ residual block outputs
(\texttt{layer2}, \texttt{layer3}, \texttt{layer4}) weighted 0.30, 0.30, 0.40.
All inputs remain native $3\times 32\times 32$ images with dataset-specific
CIFAR normalization; no ImageNet pretraining or 224x224 upsampling is used.

\paragraph{PRISM calibration.}
We calibrate thresholds using $n_\text{cal}=2{,}000$ held-out clean images
(split conformal: $n_\text{cal}$ for threshold fitting,
$n_\text{val}=1{,}000$ for coverage verification),
with $\alpha \in \{0.10, 0.03, 0.005\}$.
Thresholds and validation FPRs are generated by the artifact pipeline and
reported from the resulting JSON files rather than manually transcribed.

\paragraph{Attacks.}
We evaluate four attack classes:
\begin{itemize}[leftmargin=*,noitemsep]
  \item \textbf{FGSM} \cite{goodfellow2015explaining}: single-step gradient attack, $\varepsilon=8/255$ ($\ell_\infty$).
  \item \textbf{PGD-40} \cite{madry2017towards}: 40-step projected gradient descent, $\varepsilon=8/255$, step size $\alpha=\varepsilon/4$ ($\ell_\infty$).
  \item \textbf{CW-L2} \cite{carlini2017adversarial}: optimization-based attack, 40 iterations and 5 binary-search steps in the Vast.ai gate run.
  \item \textbf{Square Attack} \cite{andriushchenko2020square}: black-box score-based attack, 5000 queries, $\varepsilon=8/255$ ($\ell_\infty$).
  \item \textbf{AutoAttack} \cite{croce2020reliable}: standard ensemble evaluation.
  \item \textbf{Adaptive PGD}: BPDA-style activation-matching attack with $\lambda$ sweep, 100 steps, 10 restarts, and EOT verification metadata.
\end{itemize}
All attacks are implemented via Adversarial Robustness Toolbox (ART) v1.20 
\cite{nicolae2018adversarial} or native PyTorch implementations in pixel
$[0,1]$ space; \prism receives CIFAR-normalized inputs after adversarial generation.

\paragraph{Baselines.}
We reproduce four matched-FPR baselines on the same splits:
\textbf{LID} \cite{ma2018characterizing}, \textbf{Mahalanobis}
\cite{lee2018simple}, \textbf{ODIN} \cite{liang2018enhancing}, and
\textbf{Energy} \cite{liu2020energy}.

\subsection{Ablation Study}
\label{sec:ablation}

We ablate four IID detection configurations: (1)~Full \prism;
(2)~No-MoE; (3)~Ensemble-no-TDA, which removes persistence features while
keeping DCT and softmax-entropy channels; and (4)~TDA-only with its own
base-Wasserstein conformal calibrator. SACD/L0 is evaluated separately in
campaign streams because it is a temporal mechanism, not an IID detector.

% ── Campaign detection ───────────────────────────────────────────────────
\subsection{Campaign Detection (SACD)}
\label{sec:campaign}

We evaluate L0 campaign detection on clean-only, sustained, burst, and
low-rate stream scenarios generated by \texttt{run\_campaign\_eval.py}.
The C3 gate is the ASR reduction on the sustained $\rho=1.0$ stream with
clean-only L0 activation fraction at or below 1\%.

\paragraph{Practical note.}
L0 false alarm rate depends on deployment window length.
For 50-query clean windows (warmup$=35$, 15 at-risk steps),
empirical false alarm probability is $< 5\%$.
For longer windows, periodic L0 reset or auto-deactivation on
return to clean score baseline is recommended.

\subsection{Calibration Sensitivity}
\label{sec:calibration}

Figure~\ref{fig:calib} sweeps $n_\text{cal} \in \{500, 1000, 2000, 3000\}$
and confirms threshold stability: the L3 threshold varies by $\pm 0.09$
(range 0.19) across calibration set sizes, and the FPR guarantee of
Proposition~\ref{prop:coverage} holds throughout.
